## 结论

本轮评测说明：

- 当前 RAG 系统的生成阶段表现不错。
- 只要 sources 正确，回答基本能基于资料生成。
- 当前主要短板不是 LLM 回答，而是 retrieval 召回。
- 很多失败不是模型乱答，而是系统没有把正确资料送进 prompt。
- 后续优化重点应该放在 chunk 策略、top_k、相似度阈值、query 改写和检索评测上。

## 后续优化方向

| 优化方向 | 目的 |
|---|---|
| 降低相似度阈值 | 减少相关资料被过滤的问题 |
| 调大 top_k | 让更多候选 chunk 进入上下文 |
| 优化 chunk size 和 overlap | 避免关键信息被切断 |
| 按标题或段落切分 | 提高 chunk 语义完整性 |
| 增加 query rewrite | 提升同义问法的召回能力 |
| 对 retrieval 单独评测 | 区分是检索失败还是生成失败 |
